Addon File Structure
The expected folder layout and required files for any Ascension addon.

Required layout

Every Ascension addon must follow this directory structure. Ascension's loader will reject addons that are missing required files.

my-addon/
├── addon.json          # Addon manifest (required)
├── main.[ext]          # Entry point (required)
├── items/
│   └── my_item.json    # Custom item definitions
├── assets/
│   ├── textures/       # PNG textures for your items/entities
│   └── lang/
│       └── en_us.json  # Localisation strings
└── README.md

addon.json

The manifest tells Ascension who made the addon and what version of the API it targets.

{
  "id": "my-addon",
  "name": "My Ascension Addon",
  "version": "1.0.0",
  "author": "YourName",
  "ascension_api": ">=1.0.0",
  "description": "A short description of what this addon does."
}
FieldRequiredDescription
idYesUnique identifier. Use lowercase letters and hyphens only.
nameYesHuman-readable display name.
versionYesSemver string (e.g. 1.0.0).
ascension_apiYesMinimum Ascension API version required.
authorNoYour name or username.
descriptionNoShort description shown in the mod menu.

API Reference
Hooks, events, and utilities exposed by Ascension for addon developers.
ℹ️
Note All API calls go through the Ascension namespace. Import it at the top of your entry file.

Events

Subscribe to events to react to things happening in the game world. All event listeners receive a single context object.

-- Subscribe to an event
Ascension.on("item.use", function(ctx)
  -- ctx.player  → the player who used the item
  -- ctx.item    → the item that was used
  -- ctx.cancel  → call to cancel the default behaviour
  print("Player used: " .. ctx.item.id)
end)
EventWhen it firesContext fields
item.usePlayer uses any itemplayer, item, cancel
item.pickupPlayer picks up an itemplayer, item
entity.spawnAny entity spawnsentity, position
player.levelupPlayer gains a levelplayer, newLevel

Registering custom items

Ascension.registerItem({
  id      = "my_addon:cool_sword",
  name    = "Cool Sword",
  texture = "assets/textures/cool_sword.png",
  rarity  = "rare",
  onUse   = function(ctx)
    -- Called when the player uses this item
  end
})

Example Addon
A minimal but complete working addon to use as a starting point.

What this example does

This walkthrough builds a tiny addon that adds one custom item and prints a message whenever a player picks it up. It's intentionally simple — the goal is to show the full lifecycle of an addon in one place.

addon.json

{
  "id": "hello-ascension",
  "name": "Hello Ascension",
  "version": "1.0.0",
  "author": "YourName",
  "ascension_api": ">=1.0.0"
}

main.lua (entry point)

-- Register our custom item
Ascension.registerItem({
  id      = "hello-ascension:star_shard",
  name    = "Star Shard",
  texture = "assets/textures/star_shard.png",
  rarity  = "uncommon",
})

-- Listen for pickup events on our item
Ascension.on("item.pickup", function(ctx)
  if ctx.item.id == "hello-ascension:star_shard" then
    print("✨ " .. ctx.player.name .. " picked up a Star Shard!")
  end
end)
🎉
That's all it takes! Drop the folder into your addons directory, launch the game, and your item will appear. From here you can add crafting recipes, more events, custom entities, and more — see the API Reference for the full list.
