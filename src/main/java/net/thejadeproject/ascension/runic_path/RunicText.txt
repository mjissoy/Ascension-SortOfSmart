Implemented Classes:
    Rune = defines a single runic path rune, including its identity, realm/category, display data, and effect metadata.
    Runes = acts as the central registry/access point for all defined rune instances and lookup by ResourceLocation.
    RunicRuneData = stores a player’s unlocked runes and currently selected runes per major realm, and handles reading/writing that data to NBT.
    SyncRunes = syncs unlocked and selected rune data from the server to the client for UI/display purposes.
    ClientRunicData = holds the client-side cached copy of unlocked and selected runes so the GUI can render runic information without querying server-only state.
    RealmRuneSelection = represents one major realm’s selected rune list for packet syncing.
    RunicRealm = defines the different major rune categories/realms, such as Flesh, Soul, Spark, and Void.
    RuneEffectType = categorizes what kind of effect a rune provides, such as a stat boost or a passive-skill-style effect.
    RunicPathHelper = provides the core utility logic for rune data access, selection checks, slot handling, and refreshing rune-based passive skills.
    RunicScalingHelper = contains shared helper methods for scaling rune effects based on Runic path progression, especially major realm level.
    RunicTechnique = extends the generic technique system with Runic-specific behavior such as Runic cultivation skill setup, passive rune refreshes, max realm limits, per-realm rune slot counts, and rune-selection rules.
    RunicCultivationSkill = handles the active cultivation behavior for the Runic path, including applying Runic-specific cultivation gain patterns and progression through Runic realms.
    RunicCultivationSkillData = stores persistent/runtime data needed by the Runic cultivation skill, such as tracked Runic cultivation progress values specific to that skill.
    RunicArmorSkill = applies the Armor rune’s passive defensive bonuses through direct armor and armor toughness modifiers that scale with Runic progression.
    RunicStrengthSkill = applies the Strength rune’s passive bonus by increasing the custom Strength stat, scaling with Runic progression.
    RunicVitalitySkill = applies the Vitality rune’s passive bonus by increasing the custom Vitality stat, which indirectly improves survivability through derived stat-to-attribute scaling.
    RuneItem = represents rune unlock items that unlock a rune, auto-select it if a slot is available, and later allow toggling its selection while syncing and refreshing the player’s active rune state.
    RunicCultivationBoostSkill = a passive Soul rune skill that enables a global increase to cultivation gain while active.
    CultivationModifierHelper = a central utility that calculates and applies all active multipliers affecting cultivation gain across every path.
    AbstractRunicPassiveSkill = a shared base class for runic passive skills that centralizes common passive-skill boilerplate, lifecycle handling, descriptions, and icon support.
    RunicHealthRegenSkill = a passive Soul rune skill that grants continuous health regeneration over time
    HealthRegenHelper = a central utility that calculates passive health regeneration from active runic effects and is applied through a server-side tick handler.
    RunicPassiveTickHandler = a server-side tick event handler that applies ongoing runic passive effects, such as health regeneration, under the correct conditions.
    CultivationConditions = provides predefined implementations of common cultivation conditions (such as ocean-only, height-based etc) that can be reused across multiple Runic techniques.
    CultivationCondition = defines a reusable rule that determines whether cultivation is allowed and how much it is modified based on the player’s current state (environment, conditions, etc).
